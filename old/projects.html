Digital clock based on Microchip PIC16F627 microcontroller
When I was in high school, I built a four digit wall clock based on a microcontroller. I drew the PCB design using xfig, printed in on a transparency, photographed with a household UV light on a board sprayed with photosensitive material, etched in the bathroom, and drilled the holes and soldered the parts on my desk. The clock is still runing, using a wall DC adapter. When the power goes out, the controller keeps track of the time using a built-in battery, without powering the LEDs. I wrote the control program in Assembly. It supports 12/24 hour formats, and dimming using pulse width modulation.

Low-budget remote position sensor for classroom demonstation of mechanics experiments
Another high school project was inspired by an off-the-shelf educational equipment called V-Scope. The V-Scope is able to measure the position of a small button attached to a mechanical experiment (like a pendulum) using three receiving towers. The system periodically sends infrared signals to the button, which in turn emits ultrasound. By measuring how long it takes the sound to reach each tower, the system calculates the three-dimensional position of the button.
I developed a similar system, one dimensional only, that fits in the budget of Hungarian high schools. It is a quartz crystal based beeping device, which is recorded by a microphone attached to a computer. Such a device can be put together for less than one hundredth of the price of a V-Scope. I wrote a program to measure the phase shift of the beep in small time windows, and thus calculate the displacement with respect to the previous window. A small error in the frequency translates into a virtual velocity component, which can be fitted for and removed.
The first question I get is that if the device is moving, the beep gets Doppler-shifted, therefore the software will not be able to pick it up. Unfortunately I was not able to answer this question convincingly in high school, except by demonstrating that the device really worked. However, now I understand that by choosing a narrow time window, that is, multiplying the input signal with a short top-hat window function, what happens in Fourier-space is that the spectrum gets convolved with the Fourier-transform of the window function. That is, the shorter the time window, the more spread out the frequency. We just need to tune the window to be narrow enough to accomodate the highest velocity in the experiment, but wide enough not to yield too noisy results.

Object-oriented chomp
Chomp is a poset game in which two players take turns breaking off a rectangular piece from a bar of chocolate. The one left with the bottom right corner loses. I recently implemented this game in Python as an excercise in object-oriented programming: source code, screenshot.
